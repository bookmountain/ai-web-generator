## Development Constraints

1) Component design: Strictly follow the single responsibility principle, with strong reusability and maintainability.
2) API style: Prefer the Composition API and use `<script setup>` syntax appropriately.
3) Style standards: Use native CSS for responsive design, supporting desktop, tablet, and mobile layouts.
4) Code quality: Keep code concise and readable, avoid excessive comments, and prioritize complete functionality and polished styling.
5) Do not use any state management libraries, type validation libraries, or code formatting libraries.
6) Runnable output is the top priority for project generation. Use the simplest approach possible to meet requirements and avoid overly complex technologies or logic.

## Reference Configuration

1) `vite.config.js` must configure a `base` path to support subpath deployment, support importing via `@`, and must not configure a port number.


import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  base: './',
  plugins: [vue()],
  resolve: {
    alias: {
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  }
})


2) Router configuration must use hash mode to avoid server-side routing configuration issues.

import { createRouter, createWebHashHistory } from 'vue-router'

const router = createRouter({
  history: createWebHashHistory(),
  routes: [
    // Router configuration
  ]
})


3) `package.json` reference:

{
  "scripts": {
    "dev": "vite",
    "build": "vite build"
  },
  "dependencies": {
    "vue": "^3.3.4",
    "vue-router": "^4.2.4"
  },
  "devDependencies": {
    "@vitejs/plugin-vue": "^4.2.3",
    "vite": "^4.4.5"
  }
}
